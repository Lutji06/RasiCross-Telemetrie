# Phase 25: Automatisches Update (electron-updater + GitHub Releases) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Die installierte App (NSIS) prüft beim Start auf neue GitHub-Releases, lädt sie im Hintergrund und installiert beim Beenden bzw. auf Knopfdruck — mit Status-Anzeige in den Einstellungen.

**Architecture:** `electron-updater` (Runtime-Dependency) im Main-Prozess, GitHub-Provider (Repo ist öffentlich → kein Token nötig). Status fließt per IPC (`rasi-update:status`, Muster wie `serial:line`) in den Renderer; Buttons „Nach Updates suchen"/„Neustart & installieren" in der Daten-&-Backup-Gruppe. Guards: kein Update im Dev-Modus (`!app.isPackaged`), keins in der Portable-EXE (`PORTABLE_EXECUTABLE_FILE`), Browser ohne Bridge = No-op. `autoInstallOnAppQuit` + expliziter `quitAndInstall`-Button; der bestehende before-quit-Handler ist kompatibel (Installer wird vor dem Quit gespawnt).

**Tech Stack:** electron-updater, electron-builder `publish`-Config (GitHub), contextBridge-IPC.

## Locked Decisions

1. **Check nur beim Start** (nach 5 s) + manueller Button — kein Polling; Download automatisch im Hintergrund, Installation NIE während der Session (Race-Day-sicher), nur beim Beenden oder explizit.
2. **Nur Win/NSIS aktiv getestet**; mac (zip) und Linux (AppImage) erben die Publish-Config, bleiben aber unverifiziert.
3. **Unsigniert:** keine Signatur-Validierung möglich; SmartScreen-Hinweis beim Erst-Setup bleibt wie bisher.
4. **Release-Rezept ins README:** Version bumpen → `electron-builder --win --x64 --publish always` mit `GH_TOKEN` → Release mit Setup + `latest.yml`. Ohne publiziertes Release ist der Check ein sauberer „Fehler/kein Release"-Status.
