# Phase 42 — Vite + ESM-Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Alle Renderer-Scripts werden ES-Module unter `src/` mit expliziten Imports/Exports; Vite baut nach `dist/`, electron-builder paketiert `dist/` statt loser Dateien; three.js kommt als npm-Paket; die manuellen ESLint-Global-Listen entfallen.

**Architecture:** Mechanische Migration entlang der Spec-Schichtung (pure Logik → State → Dienste → UI → Einstiegspunkt). Die HTML wird zu `index.html` (Vite-Input) mit einem einzigen `<script type="module" src="/src/app.js">`; `src/app.js` importiert alle Module in der bisherigen Script-Tag-Reihenfolge und stellt eine explizite Test-/Quit-Brücke (`window.RasiTest`, `window.saveData`) bereit. main.js/preload.js bleiben CommonJS; Dev lädt `VITE_DEV_SERVER_URL`, Prod `dist/index.html`.

**Tech Stack:** Vite (neu, devDependency), three (neu, dependency, ersetzt `vendor/`), Electron 36, `@playwright/test`-Smoke-Suite aus Phase 41 als Gate, node:test.

## Locked Decisions

- **`index.html` im Repo-Root** (Vite-Default-Input): `git mv RasiCross_Telemetry.html index.html`. Alle 28 `<script src>`-Tags (Zeilen 3788–3826) entfallen zugunsten EINES Modul-Tags am selben Ort (nach `#kartEditPopover`). Der `#kartEditPopover`-Block (bisher zwischen den Tags) bleibt im Body erhalten.
- **`src/app.js` als Entry** (nicht `main.js` — Namenskollision mit Electron-Main). Importiert alle Module in exakt der bisherigen Tag-Reihenfolge und ruft nichts auf — `rasicross.js` bootet sich weiter selbst (Top-Level `init()`, Z. 1500; läuft nach Auswertung aller Imports, DOM ist da).
- **`tiles.js` bleibt UMD/CJS im Root:** main.js lädt es per `require('./tiles.js')` (Z. 6); der Renderer importiert es via Vite-CJS-Interop (`import RasiTiles from '../tiles.js'`). Einzige geteilte Datei — NICHT nach src/ verschieben.
- **Modul-Typ-Scoping statt Root-Umstellung:** `src/package.json` und `test/package.json` mit `{"type":"module"}`. Root bleibt `commonjs` → main.js/preload.js/eslint.config.js/playwright.config.js unverändert CJS. Testdateinamen bleiben.
- **Exportstil:** Interface = die bisherigen ESLint-Global-Listen (eslint.config.js dokumentiert pro Datei exakt, was sie bereitstellt). Funktions-Scripts bekommen einen benannten `export { … }`-Block am Dateiende; UMD-Objekt-Module (geo, replay, lap-engine, kart-rank, kart-registry, karts3d, drift, attitude, tile-renderer, rec-store, settings, engine, dom-targets, kart-bar, kart-overview) ersetzen ihren UMD-Footer durch benannte Exports bzw. `export default api` — Konsumenten importieren unter den bisherigen Namen (`import * as geo` bzw. `import RasiDrift from …`). `window.*`-Zuweisungen entfallen; `if (window.RasiX)`-Guards werden zu direkten Aufrufen des Imports.
- **ESLint als Migrations-Orakel:** Pro konvertierter Datei listet `npx eslint src/<datei>` (sourceType module, KEINE Interface-Globals mehr) jede fehlende Import-Referenz als `no-undef`. Migration einer Datei ist fertig, wenn sie lint-clean ist.
- **three.js:** `npm i three`; in karts3d.js `import * as THREE from 'three';` + `import { GLTFLoader } from 'three/addons/loaders/GLTFLoader.js';`, Verwendungen `new THREE.GLTFLoader(...)` → `new GLTFLoader(...)`. `vendor/` wird komplett gelöscht (Rollback laut Spec: Revert des Merge-Commits stellt vendor/ wieder her).
- **Ausgabe-Verzeichnisse:** Vite baut nach `dist/` (Spec-Wortlaut); electron-builder-`directories.output` wechselt von `dist` auf `release/` (Kollision sonst). `.gitignore`: `release/` ergänzen (`dist/` steht schon drin). `BUILD_EXE.ps1`-Hinweistexte (Z. 51/55) auf `release\` anpassen.
- **main.js-Loader:** `process.env.VITE_DEV_SERVER_URL` gesetzt → `loadURL`, sonst `loadFile(path.join(__dirname, "dist", "index.html"))`. Kein `app.isPackaged`-Sonderfall nötig — auch unpackaged lädt dist/ (deshalb bauen `start`/`test:e2e` vorher).
- **npm-Scripts:** `dev`: `vite` (HMR; Electron dazu in zweitem Terminal via `$env:VITE_DEV_SERVER_URL='http://localhost:5173'; electron .`); `start`: `vite build && electron .`; `test:e2e`: `vite build && playwright test`; alle `build*`-Scripts bekommen `vite build && `-Präfix.
- **Test-/Quit-Brücke:** ESM-Funktionen sind keine window-Globals mehr. `src/app.js` setzt explizit: `window.saveData` (der before-quit-Hook in main.js:470 ruft `saveData()` per executeJavaScript — würde sonst still brechen!) und `window.RasiTest = { state, activeRace, toggleRaceRun, endRace, activeKart, buildRaceDataForKart, RasiReplay, enterReplay, exitReplay, armRecording }`. Die e2e-Specs stellen ihre evaluate-Callbacks auf `RasiTest.*` um (einmalige Anpassung; DOM-Klick-Pfade bleiben).
- **build-manifest.test.js wird neu ausgerichtet:** alter Guard (jedes `<script src>` in `build.files`) ist gegenstandslos (ein Modul-Tag, Vite bündelt). Neuer Guard: `build.files` enthält exakt `main.js`, `preload.js`, `tiles.js`, `dist/**`, `icon.ico`; `index.html` enthält genau EIN `<script type="module">` und KEIN klassisches `<script src>`.
- **CI (`check.yml`):** js-Job: `node --check` nur noch für `main.js preload.js tiles.js` (ESM-Syntax prüft ESLint über alle src-Dateien); smoke-Job: Playwright-Step wird `xvfb-run --auto-servernum -- npm run test:e2e` (baut vorher). Beide Jobs sonst unverändert.
- **Zyklen:** ESM-Zyklen (rasicross ⇄ UI-Module) sind zulässig, weil alle Split-Scripts nur Deklarationen auf Top-Level haben (dokumentierte Konvention) und Aufrufe erst nach `init()` erfolgen. Bricht ein Zyklus doch (Smoke rot, `ReferenceError … before initialization`): die betroffene Konstante in ein eigenes Modul ziehen (Spec-Gegenmittel), nicht per Re-Export verstecken.
